# Code Of Conduct

Project discussions should remain technical, specific, and respectful.

Participants are expected to:

- Critique code and behavior rather than people.
- Provide enough evidence for claims about security, correctness, or provider
  behavior.
- Respect private information and responsible disclosure boundaries.
- Avoid harassment, discrimination, personal attacks, and sustained disruption.
- Follow maintainer requests that keep an issue or pull request focused.

Maintainers may edit, hide, lock, or remove contributions that violate these
expectations and may restrict participation for repeated or serious violations.

Report conduct concerns privately through the maintainer's GitHub profile. Do
not use public project issues to disclose sensitive personal information.
